# Contributing to AI Launcher

First off, thanks for taking the time to contribute! 🎉

AI Launcher is a small, local-first project. We keep the
process simple on purpose — no committee, no checklist of doom.

---

## 🐛 Found a bug?

1. Check the [existing issues](https://github.com/Caerleus/ai-launcher/issues)
   to see if it's already reported.
2. If not, open a new issue with:
   - **What you expected** to happen
   - **What actually happened** (error message, screenshot if useful)
   - **Your setup**: OS, Python version, GPU model
   - **Steps to reproduce** (the shorter, the better)

---

## 💡 Have an idea?

Open an issue with the `enhancement` label and describe:
- The problem you're trying to solve
- Your proposed solution (or "I don't know yet, let's discuss")

Small ideas welcome. Huge ideas welcome too — but understand
the project is intentionally minimal. Some good ideas may be
rejected just because they don't fit the "minimal launcher" spirit.

---

## 🔧 Want to send a pull request?

Awesome. Here's the path of least resistance:

### 1. Setup

```bash
git clone https://github.com/Caerleus/ai-launcher.git
cd ai-launcher
run_launcher.bat        # creates .venv + installs deps

2. Make your changes
- Keep PRs small and focused. One bug = one PR. One feature = one PR.
- Match the existing style. The project uses:
        - Python: 4 spaces, type hints where useful, ruff + black
        - JavaScript: 2 spaces, vanilla JS (no framework, no build step)
        - Markdown: keep it readable in raw form

3. Run the tests
.venv\Scripts\activate
pytest

All tests must pass. If you add a feature, add at least one test. If you fix a bug, add a regression test.

4. Commit
Use clear commit messages. Prefixes help:

fix: for bug fixes
feat: for new features
docs: for documentation
test: for tests
refactor: for code cleanup with no behavior change
Example: fix: prevent crash when models/ folder is missing

5. Open the PR
Reference the issue it solves (e.g. Closes #42)
Briefly describe what changed and why
Wait for review. Expect feedback. Don't take it personally — it's about the code.
✅ What we welcome
🐛 Bug fixes (especially on Windows + Vulkan)
📚 Documentation improvements (typos, clarifications, examples)
🧪 More tests
🔒 Security hardening (path validation, input sanitization)
🐧 Cross-platform support (macOS / Linux) — planned for v1.2
⚠️ What we probably won't merge
☁️ Cloud integrations (telemetry, sync, "AI features powered by X")
📦 Bundled models (legal + size reasons)
🎨 Major UI redesigns (the current minimal UI is a feature, not a bug)
🧱 Adding heavy dependencies (Electron, React, etc.)
If you're unsure, open an issue first to discuss before writing code. Saves everyone time.

🤝 Code of Conduct
Be respectful. Disagree without being a jerk. Help newcomers. That's it.
