## 🧭 Design Principles

- **100% local** — no cloud, no telemetry, no external calls.  
- **Zero friction** — one click to launch everything.  
- **Minimal UI** — no clutter, no distractions.  
- **Transparent** — simple `config.json`, no hidden logic.  
- **Extensible** — easy to modify, fork, or integrate.  
- **Respect hardware constraints** — designed to run even on low‑VRAM GPUs.

## ❓ Why AI Launcher?

Because running local AI shouldn’t require:

- memorizing command-line arguments  
- juggling multiple terminals  
- editing scripts every time you switch a model  
- dealing with complex UI frameworks  
- relying on cloud-based tools  

**AI Launcher gives you:**

- a single place to manage LLMs and Stable Diffusion

- real-time status monitoring

## 📦 Installation

> ⚠️ **Note on models**: AI Launcher does not include any model.
> You are responsible for downloading models you have the right to use.
> Recommended sources: [HuggingFace](https://huggingface.co), [TheBloke quants](https://huggingface.co/TheBloke).

### ⊞ Windows (recommended)

```bash
git clone https://github.com/Caerleus/ai-launcher.git
cd ai-launcher
run_launcher.bat
```

That's it. `run_launcher.bat` is **self-healing**:

- Detects Python 3.9+
- Creates a local virtual environment (`.venv/`)
- Installs dependencies from `requirements.txt`
- Starts the Flask backend and opens a native PyWebView window

If you accidentally delete `.venv/`, just relaunch the script — it rebuilds itself automatically.

### 🐧 Linux / macOS (manual)

Cross-platform support is planned for v1.2. For now:

```bash
git clone https://github.com/Caerleus/ai-launcher.git
cd ai-launcher
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cd server && python app.py
```

## 🖥️ Supported Hardware

AI Launcher **v1.1** is tested and optimized for:

| Hardware | VRAM | Status | Notes |
|----------|------|--------|-------|
| **AMD RX 580** | 4 GB | ✅ Tested | SD via `--vae-tiling` (default ON) |
| **AMD RX 6800** | 16 GB | ✅ Tested | Full SD pipeline, larger LLMs |
| **Multi-GPU AMD** | — | ✅ Supported | Select via `GGML_VK_VISIBLE_DEVICES` |
| **CPU-only mode** | — | ✅ Supported | Slow but always works |
| **NVIDIA / CUDA** | — | ❌ Not supported | Excluded by design (target: AMD low-budget) |

**Backend:** Vulkan only — CUDA is intentionally excluded.
The target audience is users with affordable AMD hardware.

> environment variable automatically. No manual config needed.

❓ FAQ
1. Launch & Backend
Backend not accessible
Make sure run_launcher.bat started Flask correctly.

Start button blocked or showing “stale model”
Starting from v1.1, the Start button performs validation before launching:

A model must be selected for each program

The selected model must still exist on disk

Models are reloaded from disk before launch

If you see “Model is no longer available. Reset to default. Please retry.”, simply press Start again — the defensive layer auto‑resets to the first valid model.
This usually happens if you rename or delete a model while the app is running.

Port 5001 / 5002 / 5003 already in use
Find the process blocking the port: 
netstat -ano | findstr :5001

Then kill the PID.
The launcher also auto‑kills stale processes on the configured ports.

PyWebView window does not open
Open the UI manually at:
http://127.0.0.1:5000  
The app gracefully falls back to your browser if PyWebView fails.

2. Models
Where are my models stored
Default structure:
models/
├── chat/      # Chat LLM (.gguf)
├── coding/    # Coding LLM (.gguf)
├── sd/        # Stable Diffusion (.safetensors / .ckpt)
├── lora/      # LoRA adapters (auto-loaded by sd-server)
└── vae/       # Optional VAE files

No models displayed
Check your paths in config.json.

Using models from another drive
Yes — use Windows directory symlinks to avoid copying gigabytes:
mklink /D C:\AI-Launcher\models\chat   D:\AI-Models\chat
mklink /D C:\AI-Launcher\models\sd     D:\AI-Models\sd

LoRA not applied
sd-server automatically loads all LoRA files from models/lora/.

3. Stable Diffusion & GPU
Vulkan OOM errors
Common on 4 GB GPUs.
AI Launcher v1.1 enables --vae-tiling automatically.

If the issue persists:

Ensure vae_tiling: true in server/config.json → images.extra

Close GPU‑heavy apps (browser, Discord, DWM)

Try CPU mode (slow but always works)

If you have a second GPU, select it from the dropdown

How to choose which GPU runs Stable Diffusion
Stable Diffusion does not support a --device flag.
AI Launcher injects the GGML_VK_VISIBLE_DEVICES environment variable based on your UI selection.
No manual configuration required.

Why do I get blank / white images
Possible causes:

VRAM borderline: 4 GB GPUs may fail during VAE decode

Out‑of‑distribution prompt: try a more suitable model (DreamShaper, RevAnimated, etc.)

Negative prompt helps:
blank, white, empty, nan, low quality

Corrupted VAE: try vae-ft-mse-840000-ema-pruned.ckpt from HuggingFace

4. Agents
Image Prompt Agent not responding
The Chat LLM must be running on port 5001.

Coding LLM not starting
Large coding models require multi‑GPU setups.

5. Python Environment
.venv corrupted or deleted
No worries — run_launcher.bat automatically rebuilds the virtual environment on next launch.

ModuleNotFoundError: utils when running tests
Activate the virtual environment before running pytest:
.venv\Scripts\activate
pytest
